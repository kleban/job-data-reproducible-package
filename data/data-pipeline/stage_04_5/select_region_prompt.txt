4. district

   * Otherwise, extract the district if available (e.g., "Rivne District") in English based on city name or administrative part or other info.

5. region
   * If the job is remote (is\_remote: "true"), set to "remote". But if data has any information about region or district not set remote
   * Otherwise, extract the oblast if available (e.g., "Rivne Oblast") in English based on city name or administrative part.

6. country
   * Always return the country name in English.
   * If the job is remote and the input language suggests a country based on description and language
7. latitude and longitude
   * Provide decimal string coordinates for original data if possible in order:
        1. If you have city - find coordinates for city
        1. If you have district - find coordinates for center of district
        2. if you have region find coordinates for region as center of region

   * If unknown or remote, leave both as empty strings ("").

Output Format

Return a valid JSON object with the following structure:

{
"original": "original location string",
"city": "city name or 'remote' or empty",
"district": "district name or 'remote' or empty",
"region": "region name or 'remote' or empty",
